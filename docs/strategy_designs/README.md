# 策略设计文档

本目录存放各交易策略的**设计文档**。**应用 AI 模型的策略**（MoE、LSTM）除算法/训练/数据分析外，还包含：

- **实现原理**：该模型在金融量化中的应用本质（如 Transformer 自注意力与 MoE、LSTM 门控与长短期记忆）
- **模型架构**：本策略的层级、维度与数据流（对应代码实现）
- **预测原理**：从输入序列到交易信号的完整路径
- **可识别的金融特征**：量价、市场结构、跨时间依赖等（参考 Transformer/LSTM 在量化中的能力边界）
- **优势与落地注意事项**：长序列/并行/过拟合/可解释性等

**通用三块**（所有策略）：

- **算法设计**：模型结构、输入输出、损失与推理/信号生成
- **训练过程设计**：数据划分、epoch/早停/保存、脚本入口（规则策略为回测与参数搜索）
- **数据分析设计**：特征来源、标签定义、验证集与评估、数据管道

策略报告（[reports/strategy_reports/](../reports/strategy_reports/)）中会链接到此处对应文档。

**设计总入口**：[需求分析和Feature测试设计](../需求分析和Feature测试设计.md)、[STATUS](../STATUS.md)。**建议优先阅读**：[回测与实盘_输入与处理流程对比](回测与实盘_输入与处理流程对比.md)、[大模型搜参设计方案](大模型搜参设计方案.md)。
