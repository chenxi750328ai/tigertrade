# 报告总览与使用说明（QA-R2）

各报告用途、更新频率与主要消费者，便于 QA 与例行检视。

| 报告/目录 | 用途 | 更新频率 | 主要消费者 |
|-----------|------|----------|------------|
| **algorithm_optimization_report** / 算法优化相关 | 算法效果、回测与实盘差异、空项根因、K 线/订单数据来源说明 | 每日例行或优化后 | 算法/策略、QA、产品 |
| **strategy_reports/**（如 strategy_moe_transformer.md） | 单策略效果、参数、回测明细 | 例行或策略迭代后 | 策略、QA |
| **order_log_analysis** / 订单日志分析 | 订单统计、成功/失败、与老虎后台差异；**加 `--with-backend`** 可拉取老虎后台订单与 order_log 按 order_id 对照（DFX），并输出 **DEMO 今日 0 成交原因**（未下单/全失败/错误类型） | 例行或按需 | 风控、QA |
| **backend_positions_analysis** | 后台持仓与挂单、有仓无止损止盈检测 | 例行 | 风控、运维 |
| **stability_report.md**（20h 稳定性） | 20h 运行结论、错误率、运行时长 | 稳定性流水线跑完后 | 运维、QA、发布关卡 |
| **QA评审报告_*** / **处理跟踪** | QA 评审结论与处理状态 | 评审后及处理进展 | 项目负责人、QA |

**需求/例行索引**：报告与需求 KPI、例行 7 项的对应关系见 [每日例行_效果数据说明](../每日例行_效果数据说明.md)、[项目质量与自测目标](../项目质量与自测目标.md)；KPI 与报告字段对照见需求文档与算法优化报告（R1 落实后将在报告内显式列出）。
