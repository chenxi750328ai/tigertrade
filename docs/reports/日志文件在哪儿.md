# 日志本来在哪儿（为什么仓库里「看不到」）

## 1. 结论

日志**不是没了**，而是：

- 写在**本机工作区**的 **`logs/`**、**`run/`** 等目录里；
- 这些目录在 **`.gitignore` 里被忽略**，**不会进 Git**，所以在远程仓库 / 只拉代码的环境里**看不到文件**；
- 若用 **`run_20h_demo.sh`** 启动，标准输出/错误会进 **`logs/demo_20h_时间戳.log`**（脚本已修复为「提示路径 = 实际路径」）。

## 2. 常见路径对照

| 内容 | 路径（相对 `tigertrade/` 根） | 说明 |
|------|------------------------------|------|
| DEMO 后台运行整段输出 | `logs/demo_20h_*.log` | `run_20h_demo.sh` 里 `nohup ... > ...log` |
| 业务订单 JSONL | `run/order_log.jsonl` | `order_log.log_order` 写入 |
| DFX（组合单/止损归因） | `run/dfx_execution.jsonl` | `order_log.log_dfx` |
| API 失败客服材料 | `run/api_failure_for_support.jsonl` | `log_api_failure_for_support` |
| 直接前台跑 `python src/tiger1.py ...` | **只在当前终端** | 未重定向则**不落盘**，除非自己 `tee` |

## 3. 为什么我（远程助手）会说「没有日志」

通常指：**当前对话里没附上文件内容**，且**仓库快照里没有 `logs/`、`run/`**（被 ignore），**不等于**你机器上从未产生日志。

## 4. 自查命令（在本机）

```bash
cd /home/cx/tigertrade
ls -lt logs/demo_20h_*.log 2>/dev/null | head -5
ls -la run/*.jsonl 2>/dev/null
```
