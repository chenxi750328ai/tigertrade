# 止损单与浮动盈亏说明

## 1. 收益展示：已实现 + 浮动

- **今日已实现盈亏**：来自当日已成交订单上的 `realized_pnl` 汇总（平仓/减仓已落袋部分）。
- **浮动盈亏**：来自 `get_positions` 返回的 `unrealized_pnl`（未平仓部分，**也是真实风险敞口**）。
- **合计快照**：`combined_pnl_usd ≈ 今日已实现 + 当前浮动`（见 `scripts/fetch_tiger_yield_for_demo.py` → `today_yield.json`）。

状态页以 `docs/today_yield.json` 为准时，请同时看 `unrealized_pnl_usd` / `combined_pnl_usd` 字段。

## 2. 为何「止损单没执行」？

### 2.1 交易所止损（STP）触发条件

- 多头止损卖单（STP）只有在行情**达到或穿过**你设置的 `stop_price` 时才会触发；**浮亏但未触及止损价**时，订单不会成交，这是正常行为。
- `DAY` 有效期的止损若遇**休市、断线、撤单**，也可能未触发。

### 2.2 设计首选：老虎 BRACKETS 组合单（限价 + 止损 + 止盈）

- SDK：`order_utils.limit_order_with_legs` + 两腿 `order_leg('LOSS', …)`、`order_leg('PROFIT', …)`，服务端将 `attach_type` 置为 **BRACKETS**（括号单：一腿成交另一腿取消）。
- 实现：`RealTradeApiAdapter.place_limit_with_bracket()`；`OrderExecutor.execute_buy` 与 `tiger1.place_tiger_order`（BUY + 限价 + 同时有 SL/TP）**优先**走组合单。
- **回退**：若 API/账户不支持组合单，再使用「限价主单 → `wait_until_buy_filled` → 单独 STP + LMT 止盈」。

### 2.3 其他

- 若日志中有 `止损单提交失败`，需看具体 API 报错（tick、合约、保证金等）。
- 旧文档 [订单执行分析_DEMO三手多单无止损单_20260202.md](./订单执行分析_DEMO三手多单无止损单_20260202.md) 描述过「仅内存止损、未挂 STP」的历史行为；当前实盘路径在买单 **FILLED** 后会尝试挂 STP。

## 3. 建议排查

1. 日志检索：`已提交止损单`、`止损单提交失败`、`超时未 FILLED`。
2. Tiger 客户端：当前是否有 **WORKING** 的 SELL STOP，是否已撤。
3. 对比止损价与当前价：浮亏但未触发多为止损价仍低于市价（多头）或尚未触及。
