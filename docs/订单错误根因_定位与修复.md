# 订单错误根因：定位与修复

## 0. 我一直用 DEMO 账户，为何统计里还有 `ALLOW_REAL_TRADING!=1`？

**代码事实**：`place_tiger_order` 里只有在 **`RUN_ENV == 'production'`**（命令行第一个参数为 **`c`** 综合账户）时才会检查 `ALLOW_REAL_TRADING`。  
**DEMO** 使用 **`python src/tiger1.py d ...`** 时 **`RUN_ENV=sandbox`**，**不会**走该分支，**不可能**因「未允许真实交易」而写这条错误。

因此汇总里出现的大量 `ALLOW_REAL_TRADING!=1`，主要来自：

1. **历史 order_log**：旧记录**没有 `run_env` 字段**，与 pytest/综合账户/一次性脚本混在一起；  
2. **曾用 `c` 或未带环境变量**启动的过程；  
3. **单元测试**里临时把 `RUN_ENV` 设为 `production` 的路径。

**从本轮起**：`order_log` 每条会带 **`run_env`**（sandbox/production），`export_order_log_and_analyze.py` 报告中有 **「ALLOW_REAL_TRADING 按 run_env 分解」** 表，可与 DEMO 区分。

## 1. 本地拒单（order_log：fail，未发老虎 API）

| 错误信息 | 根因 | 修复 |
|----------|------|------|
| **`ALLOW_REAL_TRADING!=1`**（历史统计常占大头） | 仅当 **`RUN_ENV=production`**（`python src/tiger1.py **c** ...` 综合账户）且环境变量未设时，`place_tiger_order` **故意拒绝**真实下单，避免误操作。 | **任选其一**：`export ALLOW_REAL_TRADING=1`；或在项目根复制 `.env.example` 为 `.env` 并设 `ALLOW_REAL_TRADING=1`（`tiger1` 已 `load_dotenv`）。**DEMO** 请用 `python src/tiger1.py **d** ...`，此时 `RUN_ENV=sandbox`，**不检查**该项。 |
| **持仓硬顶** | 超过 `DEMO_MAX_POSITION` / 风控上限。 | 先平仓或调参；见风控文档。 |
| **order_id 无效** | mock/测试写入 real。 | 仅真实老虎数字单记 success；见 `order_log.py`。 |

## 2. API 拒单（老虎返回错误）

| 典型错误 | 根因 | 修复 |
|----------|------|------|
| **`account` cannot be empty / 1010** | `openapicfg_*` 未配置 account 或 cwd 不对未加载到配置。 | 使用 `openapicfg_dem` 绝对路径；`run_20h_demo.sh` 已校验 account。 |
| **非交易时段 / 1200** | COMEX 休市或合约限制。 | 在交易时段重试；见老虎文档。 |

## 3. 后台今日 success=0 但本地想有成交

1. 确认启动方式：`bash scripts/run_20h_demo.sh` 或 `python scripts/run_moe_demo.py`（已设 `ALLOW_REAL_TRADING=1`）。  
2. 若用 **`c`**：必须 `.env` 或 export 见上。  
3. 若用 **`d`**：检查是否 mock 模式（`api_manager.is_mock_mode`）导致未真下单。
