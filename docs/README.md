# 文档目录说明

## 目录结构

### 核心文档
- `需求分析和Feature测试设计.md` - 项目需求和Feature测试设计
- `README.md` - 本文档

### 问题分析文档 (`lessons_learned/`)
- `订单提交问题教训记录.md` - 订单提交问题的教训
- `我的错误承认.md` - 错误承认
- `覆盖测试和下单问题回答.md` - 覆盖测试问题分析
- `Mock测试和入口文件问题分析.md` - Mock测试和入口文件问题

### 测试相关 (`reports/`)
- 测试报告、覆盖率报告等

### 指南文档 (`guides/`)
- `QUICK_START.md` - 快速开始指南
- `TEST_README.md` - 测试指南

### 其他文档
- 按日期和主题分类的各种总结、分析文档

## 文档整理建议

1. **核心文档**：保留在docs根目录
2. **问题分析**：移动到`lessons_learned/`
3. **测试报告**：移动到`reports/`
4. **临时文档**：考虑归档或删除
